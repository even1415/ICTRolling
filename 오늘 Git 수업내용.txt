1. 브랜치 개념
메인브랜치 : 전체적인 흐름/메인메소드 -> 마스터

사용자브랜치 : 유저 각자 흐름 -> (이름 설정 가능)

2. 용어 설명
클론 : 본인 Git 레퍼지토리(저장소)를 사용자 컴퓨터로 복사해오는 것
브랜치 : 레퍼지토리의 진행 상황
커밋 : 사용자 컴퓨터에서 브랜치로 업로드하는 것
푸시 : 브랜치에서 다른 브랜치 혹은 Git 레퍼지토리로 업로드하는 것
풀 : 다른 브랜치 혹은 Git 레퍼지토리에서 브랜치로 다운로드하는 것

병합(합병) : 2개 이상의 브랜치를 합치는 것
폐기(되돌리기) : 브랜치의 변경사항을 취소하는 것
페치 : 풀 받기 전에, 변경내용을 확인하고 싶은 경우(협업모드에서 주로 씀)

콜라보레이터(Collaborator) : 직접 수정을 하는 사람 -> 수정권한 있음 (팀원)
컨트리뷰터(Contributor) : 기부자, 제안을 하는 사람 -> 수정권한 없음 (제안)

충돌 : 동시에 같은 파일을 2명 이상 올리는 경우 발생(같은 파일은 2개 존재할 수 없음)
이 때는 반드시 먼저 백업 후 풀 받고, 본인 내용을 반영해서 다시 푸시해보자
